# Reports Guide

## Types of Reports

Citi Maju issues two main categories of report:

**Progress Reports** — issued during active works to update the client on status, progress against programme, and any issues.

**Completion Reports** — issued at the end of a project or phase, confirming works done, test results, and handover information.

---

## Progress Report

### When to Issue
Monthly, or as agreed with the client. Issue on a fixed date (e.g., last working day of each month) so the client can plan around it.

### Required Contents

1. **Cover page** — entity letterhead, project name, report period, report number, date
2. **Project summary** — one paragraph: project name, client, contract value, start date, completion target
3. **Progress summary** — what was done this period (bullet points, clear and factual)
4. **Programme status** — are you on time, ahead, or behind? State the reason if behind.
5. **Percentage completion** — by major work item if applicable
6. **Issues and risks** — any blockers, client actions required, or risks to the programme
7. **Next period activities** — what is planned for the coming month
8. **Site photographs** — at least three, labelled with date and description
9. **Appendices** — test records, delivery records, subcontractor programmes if applicable

### Report Reference
`[Entity]-PR-[Project Code]-[Report No.]`
Example: `CME-PR-SA01-004` (Engineering, project SA01, 4th progress report)

---

## Completion Report

### When to Issue
At practical completion of works, before or during the defects liability period.

### Required Contents

1. **Cover page** — entity letterhead, project name, "Completion Report", date
2. **Project details** — client, contract reference, contract value, start date, completion date
3. **Summary of works** — what was delivered, by item, in table format
4. **As-built record** — reference to as-built drawings submitted separately
5. **Test results** — hydrostatic pressure test results, CCTV inspection records, or other applicable tests
6. **Certificates** — SIRIM, SPAN, or authority certificates obtained during the project
7. **Handover checklist** — items handed over to client (documents, keys, equipment manuals)
8. **Defects liability period** — start date, end date, contact for defect reporting
9. **Photographs** — before, during, and after

## Report Design Standards

**Cover page:** Entity letterhead. Report title in Cormorant Garamond 400. Client name and report date below in Sora.

**Interior pages:**
- Section headings: Cormorant Garamond 400, 16–18pt (H2 level)
- Sub-headings: Sora 600, 11–12pt
- Body: Noto Sans 400, 10pt, line height 1.65
- Tables: Blue Pale (#E8F1FB) header rows, alternating White/Cream rows
- Photographs: captioned with Noto Sans 300, 8pt, Text Light

**Page footer:** Page numbers, report reference, report date — Sora 300, 7pt, centre-aligned

**Gold accent:** Hairline rule below report cover heading. Section divider hairlines.

---

## Language and Tone in Reports

Reports are factual documents. Write in the third person where convention requires (e.g., "Works were completed...") but active voice is acceptable where it reads more naturally.

**Do:**
- State facts: "Pipe laying from MH03 to MH05 completed. 145m installed."
- Quantify: "Overall programme: 68% complete as at [Date]."
- Flag issues clearly: "Access to Zone B is restricted pending client coordination. This is anticipated to delay Zone B by approximately 5 working days."

**Do not:**
- Minimise problems: "Everything is fine, just a minor delay."
- Overstate progress: Report only what is genuinely complete, not what is in progress.
- Use vague language: "Some progress was made this month."

---

## Distribution

Reports are issued as PDF to the client's designated project contact. Copy the relevant internal team members.

File naming: `[Report Ref]_[YYYY-MM-DD].pdf`
Example: `CME-PR-SA01-004_2025-04-30.pdf`

Retain a copy of all reports in the project file.
