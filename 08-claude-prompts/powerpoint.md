# Claude Prompts — PowerPoint

Use these prompts when asking Claude to create or structure a PowerPoint presentation for Citi Maju. Paste the relevant prompt, fill in the bracketed fields, and Claude will produce content that follows brand standards.

---

## Master Context Block

Paste this at the start of any Claude session involving Citi Maju PowerPoint creation:

```
You are working on a PowerPoint presentation for Citi Maju Group, a Klang Valley construction and engineering contractor. Follow these brand rules strictly:

Company: Always "Citi Maju Group" — never "two companies". Tagline: "Trusted. Certified. Delivered."
Services: Large-scale water reticulation (Engineering), renovation, pipe jacking, directional drilling, hot tapping, smaller reticulation works (Construction). Do NOT mention machinery rental.
Certifications (Engineering entity only): CIDB G7 Grade · SPAN Registered · Air Selangor Approved Contractor
Territory: Always "Klang Valley" — never "KL and Selangor"
Contact: inquiry@citimaju.com | www.citimaju.com

Typography rules for slides:
- Slide titles (H1): Cormorant Garamond 400 — NEVER Sora or sans-serif
- Sub-headings, labels, bullets: Sora
- Body copy: Noto Sans

Colour rules:
- Navy (#071E3D): Dark backgrounds
- Brand Blue (#1B4F8A): Primary headings on light backgrounds
- Gold (#C8921A): Accent hairlines ONLY — never large fills
- Text Dark (#0D1B2E): Body copy on light slides

Tone: Direct, credible, precise. No superlatives. No vague claims. No "world-class" or "comprehensive solutions".
```

## Prompt 1 — Company Introduction Deck

```
[Paste master context block above first]

Create a 10-slide company introduction presentation for Citi Maju Group. 

Audience: [Describe: e.g., "property developer project director, first meeting"]
Purpose: [e.g., "introduce the group and open discussion about upcoming water reticulation project"]

Structure the deck as follows:
1. Cover — project/meeting title, date, Citi Maju Group
2. Who We Are — one paragraph, credentials lead
3. What We Do — services, two columns (Engineering / Construction)
4. Why Trust Us — CIDB G7 explained, SPAN, Air Selangor, track record statement
5. Project References — table: description, scope type, year (no client names unless approved)
6. [Two to three slides specific to this client's likely interests — ask me what their focus is if unclear]
7. Our Approach — method, team, programme overview
8. Contact / Next Steps — inquiry@citimaju.com, www.citimaju.com, tagline

For each slide, provide:
- Slide title (in Cormorant Garamond 400 — note this in brackets)
- Eyebrow label above title (in Sora 300 ALL CAPS — note in brackets)
- Bullet points or body text
- Notes for the speaker (in [brackets] below each slide)
```

## Prompt 2 — Tender / Bid Presentation

```
[Paste master context block above first]

Create a tender presentation for Citi Maju [Engineering / Construction] for the following project:

Project: [Project name and brief description]
Client / Authority: [Client or tender authority]
Scope: [Summarise the scope of works]
Contract value (approximate): [RM value if known]
Key differentiators to emphasise: [e.g., CIDB G7, trenchless method, previous similar work]

Structure:
1. Cover — "Tender Submission: [Project Name]", entity name, date
2. Executive Summary — one-page summary: what we are proposing, why Citi Maju
3. Company Credentials — certifications, relevant track record
4. Understanding of Requirements — restate the scope in our terms, highlight key technical considerations
5. Proposed Methodology — how we will execute the works (detail the specific method)
6. Programme — indicative timeline in table or bar chart format
7. Team and Resources — key personnel, equipment (do not include unconfirmed resources)
8. Pricing Summary — as per the separate quotation (reference the quotation document)
9. Closing — why Citi Maju, contact details, tagline

Provide full slide content for slides 1–5 and 9. For slides 6–8, provide the structure and placeholders — I will fill in the specific data.
```

## Prompt 3 — Project Progress Update

```
[Paste master context block above first]

Create a monthly project progress update presentation for the following project:

Project: [Project name]
Client: [Client name or type]
Report period: [e.g., April 2025]
Report number: [e.g., Report No. 4]

Information to include:
- Overall progress: [% complete]
- Works completed this period: [List what was done]
- Programme status: [On time / X days ahead / X days behind — with reason if behind]
- Issues or risks this period: [List any]
- Client actions required: [List if any]
- Planned works next period: [List]
- Site photographs available: [Yes/No — I will insert them]

Structure the deck in 8 slides:
1. Cover — project name, period, report number
2. Project summary — one-paragraph overview of the project
3. This period — what was done (bullet list, factual)
4. Programme status — table: activity, planned %, actual %, variance
5. Issues and risks — table: issue, impact, action required, responsible party
6. Next period — planned activities (bullet list)
7. Site photographs — four photograph placeholders with caption labels
8. Contact — project manager name, mobile, email

Keep the tone factual. No padding. No "we are happy to report" — just the data.
```
